# Ignis Fatuus

## Project Description
I made an interactive Game Menu with Details and A Small Introduction about the game I would be planning to make in the future! It is styled as a visual novel. 

## Intended User or Player
This interaction is mainly targetting at teenagers and young adults as it will tackle more heavy topics but will be heavily enjoyed by this and ghost hunting lovers as well! 

## User / Player Goal
The intended player’s goal is to follow through with the story and make your own choices. These choices will matter in the end as they will provide each a unique storyline and ending. They should be able to understand the storyline through a series of reading as well as picking up how certain characters are speaking. 

## Inspiration Interfaces
1. Luigi Mansion 3 Game Menu - The one thing that stood out from all the other Luigi mansion 3 is the layout of showing the actual map instead of having just a gloomy screen with the iconic logo. In the day version of this game UI, it’s all sunshine, the lighting is bright and it looks pleasant, once you proceed with the game and pass the prologue, the screen will turn into this. I love this concept of showing the overall gloomy and horror feel to the mansion itself. The next design choice I noticed is the logo itself. Lastly is the actual buttons you click, they’re a completely different color, opting them to be easy to read. 

What I would like to implement from this game UI is the overall mood and the colors and lighting that was chosen to represent this game and its theme of ghosts, which is very similar to my own theme. 

2. Homicipher Game Menu - This is a horror game that is themed as a visual novel. The first element you see here is the ominous character that is looming over the title screen. While this game is in Japanese, you can see the title has both Kanji (Japanese) as well as English below it to relay it to its vast international user group. Another component in design making I noticed is the lack of options, there are some icons on the bottom right but the only options you see is either to create a new game save or to load a pre-existing one. I think this is for the purpose of simplicity but it’s effective. 

The biggest inspiration I will use here is the character screen idea. If you research more into this game, there are actually different screens containing different characters and I love that idea completely. 

3. IB Game Menu - This is very similar to my own wireframe and the idea. You notice very dull yet simple color choices that make the main character stand out. There is a completely black and white use so that it is very readable and accessible to see for its user audience. When you hover over the options, they turn from Grey to White, providing good feedback status. I love the sketchy feel that this game menu has from the painting to the actual character itself. In the actual game itself it has a small animation to it that really stands out from the status background. 

That would be one of the ideas I should implement to make sure the user group knows what they are clicking and it isn’t just a dull color not standing out. 

## Design Choices
- Layout: The layout predominantly showcases the art on the right side of the screen then itll focus on the interfaces on the left size of the "game menu." There will be two icons all the way at the botton and 3 interactive buttons which provide feedback as when you are hovering over it. 
- Color: The color scheme that was went went consistented of a lot of saturated and muted blues as well as a moonochrome with black and white respectively. 
- Typography: The main typography used was Corvid, which is a gothic styled cursive text family. 
- Visual hierarchy: As discussed earlier the visual hierarchy lands in the artwork itself then the visual UI on the left side of the screen. This will be fixed everytime you widen a screen or make a screen full. 
- Feedback states: When you hover over each icon, a button will light up, itll be lighter than the actual text and box around it to indicate you are getting teh feedback needed. 
- Responsive design: The responsive design is for fun, I actually implemented some links that have been shown to the public before as well as a link to join for an upcoming project that has been worked on for years
